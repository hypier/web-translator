# 欢迎贡献者

您好！感谢您对为 LangChain 贡献感兴趣。作为一个在快速发展的领域中的开源项目，我们非常欢迎各种贡献，无论是新功能、基础设施改进、文档优化还是错误修复。

## 🗺️ 指南

### 👩‍💻 贡献的方式

有很多方式可以为 LangChain 贡献。以下是一些常见的贡献方式：

- [**文档**](/docs/contributing/documentation/): 帮助改善我们的文档，包括这份文档！
- [**代码**](/docs/contributing/code/): 帮助我们编写代码、修复错误或改善我们的基础设施。
- [**集成**](integrations.mdx): 帮助我们与您最喜欢的供应商和工具进行集成。
- [**讨论**](https://github.com/langchain-ai/langchain/discussions): 帮助回答使用问题并与用户讨论问题。

### 🚩 GitHub Issues

我们的 [issues](https://github.com/langchain-ai/langchain/issues) 页面会及时更新错误、改进和功能请求。

有一个标签分类系统可以帮助排序和发现感兴趣的问题。请使用这些标签来帮助组织问题。

如果您开始处理某个问题，请将其分配给自己。

如果您添加一个问题，请尽量将其集中在一个单一的、模块化的错误/改进/功能上。如果两个问题相关或相互阻塞，请将它们链接在一起，而不是合并。

我们会尽量保持这些问题尽可能更新，不过由于该领域的快速发展，有些可能会过时。如果您注意到这种情况，请告知我们。

### 💭 GitHub 讨论

我们有一个 [讨论](https://github.com/langchain-ai/langchain/discussions) 页面，用户可以在这里提出使用问题，讨论设计决策，并提出新功能建议。

如果您能够帮助回答问题，请这样做！这将使维护者能够花更多时间专注于开发和修复错误。

### 🙋 获取帮助

我们的目标是提供尽可能简单的开发者设置。如果您在设置过程中遇到任何困难，请联系维护者！我们不仅希望帮助您解决问题，还希望确保这个过程对未来的贡献者来说是顺利的。

同样，我们在代码库中确实执行某些代码检查、格式化和文档标准。如果您发现这些难以处理（甚至只是令人烦恼），请随时联系维护者寻求帮助——我们不希望这些影响到优秀代码的提交。

### 🌟 认可

如果您的贡献已被纳入发布中，我们希望在 Twitter 上给予您认可（但仅在您愿意的情况下）！如果您有希望我们提及的 Twitter 账号，请在 PR 中或通过其他方式告知我们。
