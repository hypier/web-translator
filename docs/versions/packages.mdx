# 📕 包版本控制

目前，LangChain 采用临时的发布流程：由维护者高频率地进行发布，并发布到 [PyPI](https://pypi.org/)。
不同的包版本控制略有不同。

## `langchain-core`

`langchain-core` 当前版本为 `0.2.x`。

由于 `langchain-core` 包含整个 LangChain 生态系统的基础抽象和运行时，我们将提前通知任何重大更改并进行版本升级。唯一的例外是标记为 `beta` 装饰器的内容（您可以在 API 参考中看到这一点，并在使用此类功能时会看到警告）。引入 beta 特性的原因是，由于该领域变化的速度，能够快速响应仍然是优先事项。

小版本增加将发生在：

- 任何标记为 `beta` 的公共接口的重大更改。

补丁版本增加将发生在：

- 错误修复
- 新特性
- 对私有接口的任何更改
- 对 `beta` 特性的任何更改

## `langchain`

`langchain` 当前版本为 `0.2.x`

次版本增加将发生在：

- 任何未标记为 `beta` 的公共接口的破坏性更改。

补丁版本增加将发生在：

- 错误修复
- 新功能
- 任何对私有接口的更改
- 任何对 `beta` 功能的更改。

## `langchain-community`

`langchain-community` 当前版本为 `0.2.x`

版本将按照与 `langchain` 相同的指南进行更新。

## `langchain-experimental`

`langchain-experimental` 当前版本为 `0.0.x`

所有更改都将伴随补丁版本的增加。

## 合作伙伴套餐

合作伙伴套餐是独立版本控制的。
